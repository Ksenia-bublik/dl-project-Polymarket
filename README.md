# Вероятностное прогнозирование в домене поп-культуры

## О проекте

Этот проект посвящён вероятностному прогнозированию исходов событий в домене поп-культуры на основе реальных рынков Polymarket, синтетически сгенерированных вопросов и новостного контекста.

Итоговая цель проекта — построить воспроизводимый конвейер, который:

- собирает и очищает реальные рынки по культуре;
- расширяет покрытие домена синтетическими вопросами;
- подбирает к каждому примеру релевантный новостной контекст;
- восстанавливает историческую рыночную вероятность Polymarket как внешний ориентир;
- обучает модель предсказывать вероятность исхода `YES` только по вопросу и новостям.

Важно: в финальной рабочей версии проекта основным подходом является **news-only SFT**, то есть обучение по размеченным примерам **без подачи рыночной вероятности на вход модели**.

---

## Постановка задачи

Для каждого примера задано:

- **question** — вопрос о будущем событии;
- **prediction_date** — момент, на который как будто делается прогноз;
- **news_context** — новостной контекст, доступный до этой даты;
- **resolution_date** — дата фактического разрешения события;
- **answer / resolution** — итоговый исход (`YES/NO` или `1/0`).

Требуется предсказать вероятность положительного исхода:

$$
P(\text{YES} \mid \text{question}, \text{news\_context})
$$

Проект решает именно задачу **вероятностного прогнозирования**, а не только бинарной классификации. Поэтому ключевое внимание уделяется качеству вероятностей и их калибровке.

---

## Основная идея проекта

Проект строится вокруг трёх источников сигнала:

1. **Реальные рынки Polymarket по культуре**  
   Используются как основной источник разрешённых событий с бинарным исходом.

2. **Синтетические вопросы по поддоменам поп-культуры**  
   Нужны для расширения покрытия редких типов событий и увеличения разнообразия формулировок.

3. **Новостной контекст через Exa**  
   Для каждого вопроса подбираются новости, опубликованные **до даты прогноза**, чтобы исключить утечку будущей информации.

---

## Что в итоге является основной моделью

В проекте были рассмотрены два направления:

- ранний подход в духе **RLVR / ReMax**;
- итоговый устойчивый подход — **news-only SFT**.

На практике основной рабочий результат был получен именно через **SFT**, а не через прямое обучение по награде.

### Что это означает

В финальной версии:

- модель получает на вход **вопрос + сжатый новостной контекст**;
- рыночная вероятность `prediction_polymarket` **не подаётся во вход**;
- модель выдаёт **вероятность события `YES`**;
- после обучения вероятности дополнительно **калибруются**.

То есть текущая версия проекта — это **честная news-only модель**, а не гибрид с рыночной вероятностью.

## Полный конвейер проекта

### 1. Сбор и очистка реальных данных
Из Polymarket отбираются только:
- рынки домена pop culture;
- закрытые и разрешённые рынки;
- бинарные рынки формата `YES/NO`.

Далее проводится очистка:
- нормализация полей;
- удаление некорректных строк;
- приведение исходов к единому виду `0/1`.

### 2. Генерация синтетических вопросов
Для расширения домена генерируются дополнительные вопросы по поддоменам:
- Movies / TV;
- Music;
- Awards;
- Gaming;
- Creators.

К вопросам применяются фильтры качества:
- бинарность;
- проверяемость;
- принадлежность к домену культуры;
- отсутствие чрезмерно субъективных формулировок;
- наличие временной рамки.

### 3. Подбор новостей через Exa
Для каждого вопроса подбирается внешний контекст:
- из вопроса строится запрос в Exa;
- выбираются только новости, опубликованные **до `prediction_date`**;
- материалы дополнительно фильтруются по релевантности;
- сохраняется компактный `news_context`.

Для сложных случаев используются отдельные ноутбуки с более мягкими настройками поиска.

### 4. Сборка итогового набора данных
Все источники приводятся к единому формату:
- `question`
- `prediction_date`
- `resolution_date`
- `answer`
- `resolution_reasoning`
- `news_context`

### 5. Получение рыночного ориентира
Для каждого реального рынка внутри интервала между открытием и закрытием случайно выбирается `sampled_prediction_date`, после чего восстанавливается историческая цена YES-токена через Gamma API и CLOB API.

Это даёт **историческую рыночную вероятность** на момент прогноза.

### 6. Обучение news-only модели
На финальном этапе обучается компактная модель, которая получает:
- вопрос;
- дату закрытия;
- сжатый новостной контекст.

На выходе модель выдаёт вероятность исхода `YES`.

## Гибридный базовый ориентир

В проекте отдельно используется **гибридный базовый ориентир**, который нужен как сильная верхняя практическая планка.

Его логика:
- если в строке есть `prediction_polymarket`, используется откалиброванная рыночная вероятность;
- если её нет, используется текстовая модель.

Важно: этот ориентир **не является основной news-only моделью проекта**, потому что почти всегда опирается именно на рынок.

### Текстовая ветка ориентира
Использует:
- TF-IDF по вопросу;
- TF-IDF по очищенному тексту;
- признаки по дате;
- логарифм объёма торгов;
- логистическую регрессию;
- изотоническую калибровку.

---

## Итоговое обучение: news-only SFT

Финальный рабочий вариант проекта — это **обучение по размеченным примерам**.

### Основные характеристики
- базовая модель: `Qwen/Qwen2.5-1.5B-Instruct`;
- режим: бинарная вероятностная классификация;
- во входе: `question + compact news evidence`;
- рыночная вероятность во вход **не включается**;
- используется LoRA;
- применяется 4-битная загрузка модели;
- вероятности после обучения дополнительно калибруются.

### Почему этот подход стал основным
Потому что он оказался:
- устойчивее;
- воспроизводимее;
- проще в отладке;
- лучше подходящим для news-only постановки, чем ранний RLVR/ReMax.

---

## Метрики

В проекте основное внимание уделяется вероятностным метрикам.

### Brier score
Основная метрика качества вероятностного прогноза:

```text
Brier = mean((prediction - outcome)^2)
```

Чем ниже значение, тем лучше.

### ROC-AUC
Дополнительная метрика, показывающая, насколько хорошо модель ранжирует положительные и отрицательные примеры.

---

## Ключевой методический вывод

На этом проекте наиболее устойчивый и воспроизводимый результат дал именно **news-only SFT**, а не прямой RLVR/ReMax.

Это не означает, что RLVR неинтересен. Напротив, он остаётся содержательно важным направлением для дальнейших экспериментов. Но на текущем этапе основной рабочий подход проекта — это:

1. построить честный news-only набор данных;
2. обучить сильную модель по размеченным примерам;
3. откалибровать вероятности;
4. сравнить результат с рыночным ориентиром.

---

## Ограничения проекта

- домен pop culture шумный и неоднородный;
- качество новостного контекста зависит от полноты поиска;
- гибридный ориентир трудно превзойти, если он почти всегда использует рынок;
- компактная модель ограничена по мощности;
- прямой RLVR/ReMax на таком наборе данных оказался слишком хрупким.

## Статус проекта

Проект находится в исследовательской стадии, но уже даёт воспроизводимый рабочий результат:

- реализован честный news-only подход;
- получен сильный рыночный ориентир для сравнения;
