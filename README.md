# LLM Forecasting SFT

Проект посвящён обучению небольшой языковой модели для прогнозирования исходов событий на рынках предсказаний. Модель получает текст события и связанные с ним данные, после чего должна выдать вероятностный прогноз того, произойдёт событие или нет.

В текущей версии проекта используется **SFT — Supervised Fine-Tuning**. Методы reinforcement learning, такие как GRPO, ReMax или RLVR, в этом проекте не применяются.

## Цель проекта

Основная задача — проверить, может ли дообученная LLM давать полезные вероятностные прогнозы по событиям из Polymarket и похожих рынков предсказаний.

Модель обучается на исторических примерах вида:

- описание события;
- дополнительный контекст, если он доступен;
- фактический исход события;
- целевой ответ в формате вероятности или бинарного прогноза.

После обучения модель можно использовать для генерации прогнозов на новых событиях и сравнивать её качество с простыми бейзлайнами, например с вероятностью Polymarket.

## Используемый метод

В проекте используется **Supervised Fine-Tuning (SFT)**.

SFT — это обучение модели на готовых парах `prompt -> target response`, где модель учится воспроизводить правильный формат ответа и связывать входные признаки с итоговым прогнозом.

В отличие от RL-подходов, SFT не оптимизирует модель напрямую через reward-функцию. Вместо этого модель обучается на заранее подготовленных целевых ответах.

## Общий пайплайн

1. **Сбор данных**
   - Исторические события Polymarket.
   - Рыночные вероятности на выбранную дату.
   - Итоговый исход события.
   - Метаданные события: даты, категории, объём, ликвидность и другие признаки.

2. **Подготовка датасета**
   - Очистка и нормализация данных.
   - Удаление некорректных или нерешённых событий.
   - Приведение исходов к единому формату.
   - Формирование обучающих примеров для SFT.

3. **Формирование prompt-ов**
   - Каждый пример превращается в инструкцию для модели.
   - В prompt включается информация, доступная на момент прогноза.
   - В target-ответ помещается ожидаемый прогноз или объяснение с финальной вероятностью.

4. **SFT-обучение**
   - Используется небольшая instruct-модель.
   - Обучение выполняется через стандартный supervised fine-tuning.
   - Для экономии памяти может использоваться LoRA или QLoRA.

5. **Инференс**
   - Обученная модель прогоняется на тестовом наборе.
   - Из ответа модели извлекается вероятность.
   - Результаты сохраняются в таблицу.

6. **Оценка качества**
   - Brier Score.
   - Accuracy при бинаризации прогноза.
   - Profit / ROI в простой симуляции ставок.
   - Количество прибыльных и убыточных ставок.

## Пример формата обучающего примера

```json
{
  "instruction": "Оцени вероятность того, что событие произойдет. Ответь числом от 0 до 1.",
  "input": "Событие: Will candidate X win the election?\nДата прогноза: 2024-10-01\nРыночная вероятность Polymarket: 0.63\nКонтекст: ...",
  "output": "0.63"
}
```

Формат может отличаться в зависимости от конкретной версии ноутбука, но логика остаётся такой же: модель получает описание ситуации и учится выдавать корректный прогноз.

## Оценка

Пример запуска оценки:

```bash
python scripts/evaluate.py
```

Основные метрики:

### Brier Score

Brier Score измеряет качество вероятностного прогноза:

```text
brier = (prediction - outcome) ** 2
```

Чем ниже значение, тем лучше.

### Accuracy

Для accuracy прогноз можно бинаризовать:

```text
prediction >= 0.5 -> YES
prediction < 0.5 -> NO
```

### Profit / ROI

Для простой симуляции ставок можно считать, что стратегия покупает сторону, которую считает более вероятной.

Пример логики:

- если `prediction >= 0.5`, стратегия покупает YES;
- если `prediction < 0.5`, стратегия покупает NO;
- прибыль считается по фактическому исходу события.

Также полезно считать:

- общее количество ставок;
- количество прибыльных ставок;
- количество убыточных ставок;
- суммарную прибыль;
- ROI.

## Важные ограничения

1. **SFT не гарантирует улучшение качества прогнозов**

   Модель может научиться формату ответа, но не обязательно станет лучше прогнозировать реальные события.

2. **Нужен строгий контроль temporal leakage**

   В prompt не должна попадать информация, которая стала известна после даты прогноза.

3. **Рыночная вероятность — сильный бейзлайн**

   Polymarket уже агрегирует мнение участников рынка, поэтому обогнать его сложно.

4. **Маленькая модель может быть ограничена**

   Небольшая модель удобна для экспериментов, но её качество может быть ниже, чем у крупных моделей.

5. **Бэктест не равен реальной торговле**

   Историческая симуляция не учитывает все реальные ограничения: ликвидность, комиссии, проскальзывание, задержки и изменение рынка после ставки.

## Отличие от RL-подходов

В статье *Outcome-based Reinforcement Learning to Predict the Future* авторы обучали модели для прогнозирования через RLVR-подходы, включая GRPO, Modified-GRPO и ReMax, и сравнивали их по Brier Score, ECE и симуляции торговли на Polymarket.

В этом проекте используется другой и более простой подход: **SFT**. Поэтому результаты, гиперпараметры и выводы RL-подходов нельзя напрямую переносить на этот проект.

После успешного обучения желательно сохранить:


Если используется LoRA, обычно сохраняется не вся модель, а только adapter. Для последующего инференса нужно загрузить базовую модель и применить к ней сохранённый adapter.

## Минимальный чеклист успешного запуска

Перед тем как считать эксперимент успешным, нужно проверить:

- датасет не пустой;
- train/test split сделан корректно;
- в train не попали будущие данные из test;
- модель реально обучалась, а loss менялся;
- adapter или модель сохранились на диск;
- инференс запускается на сохранённой модели;
- ответы модели парсятся в числовую вероятность;
- результаты сохранены в файл;
- посчитаны Brier Score, Accuracy, Profit и ROI.
